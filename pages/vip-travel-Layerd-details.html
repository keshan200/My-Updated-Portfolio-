Project Overview

The VIP-Travel Layered Architecture Project is an enhanced and modularized version of the VIP-Travel Management System, which was initially developed as my first-semester final project at the Institute of Software Engineering (IJSE). This updated version employs a robust layered architecture, emphasizing scalability, maintainability, and modular design.

The system is designed to efficiently handle core car rental operations such as managing customers, users, vehicles, and bookings while ensuring a seamless experience for the end-users. The transition to a layered architecture separates the business logic, data access, and user interface into distinct layers, making the system easier to maintain and extend.

By restructuring the codebase, the project now demonstrates improved code reusability and organization, adhering to industry standards for enterprise application development.

Key Features

  • Layered Architecture with DAO, Service, and Controller layers for clear separation of concerns
  • Scalable design to support future enhancements and integrations
  • Comprehensive management of customers, vehicles, and bookings
  • Role-based access control ensuring secure user authentication
  • Dynamic report generation with Jasper Reports for invoices and operational summaries
  • Optimized database operations with MySQL/PostgreSQL
  • Intuitive user interface built using JavaFX and Scene Builder
  • Dependency and build automation handled by Maven
